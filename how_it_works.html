Retry Engine — Exponential Backoff + Jitter
When a provider call fails, ai-bridge-kit automatically retries with exponential backoff and optional jitter. Validation errors are never retried.
retry.py core logic 
# AIBridgeSettings defaults
retries          = 2
backoff_seconds  = 0.4
max_backoff      = 4.0
use_jitter       = True

# Delay formula per attempt:
delay = min(max_backoff, backoff * (2 ** (attempt-1)))
if jitter: delay *= uniform(0.75, 1.25)

# Never retry these:
should_retry = not isinstance(exc, (
  ValidationError,
  ProviderNotFoundError,
  CapabilityNotSupportedError
))

End-to-End Code Walkthrough
From install to response in under 10 lines. Every line annotated with what happens internally.
full_example.py annotated 
# Step 1 ── Install
# pip install "ai-bridge-kit[openai]"

# Step 2 ── Import
from ai_bridge_kit import AIClient
from ai_bridge_kit.providers import OpenAIProvider, AnthropicProvider, OllamaProvider

# Step 3 ── Create client (auto-registers LocalFunctionProvider as default)
client = AIClient()  # settings: timeout=30s, retries=2, jitter=True

# Step 4 ── Register a cloud provider and set as default
client.register_provider(
    OpenAIProvider(api_key="sk-..."),
    set_default=True   # stored in settings.default_provider
)

# Step 5 ── Chat  (sync or async)
resp = client.chat("Explain RAG in one sentence.", model="gpt-4o-mini")
# Internally: normalize → ChatRequest → run_with_retry → timeout guard → provider.chat()
print(resp.content)          # typed AIResponse.content  ✓
print(resp.tokens_output)    # tokens_output: int | None ✓

# Step 6 ── Embeddings
emb = client.embed(["hello", "world"])
print(len(emb.vectors[0]))   # 1536 dims (OpenAI) ✓

# Step 7 ── Switch to Ollama (local) — zero code change above
client.register_provider(
    OllamaProvider(base_url="http://localhost:11434", default_chat_model="llama3.2"),
    set_default=True
)
resp2 = client.chat("Explain RAG in one sentence.")  # same call ✓

# Step 8 ── CLI (no Python needed)
# ai-bridge chat --message "Explain RAG"
# ai-bridge embed --text "hello" --text "world"
# ai-bridge providers

Error Hierarchy — Predictable, Typed Exceptions
Every failure mode has its own exception class. You always know exactly what went wrong — and whether it should be retried.
errors.py hierarchy 
class AIBridgeError(Exception):
    """Base — catch everything"""

class ProviderNotFoundError(AIBridgeError):
    """Provider not registered — never retry"""

class ProviderExecutionError(AIBridgeError):
    """API call failed — WILL retry"""

class ValidationError(AIBridgeError):
    """Bad input — never retry"""

class CapabilityNotSupportedError(AIBridgeError):
    """Provider lacks capability — never retry"""
handling errors best practice 
from ai_bridge_kit.errors import (
    AIBridgeError,
    ProviderExecutionError,
    ValidationError,
)

try:
    resp = client.chat("Hello")
except ValidationError as e:
    print("Bad input:", e)      # not retried
except ProviderExecutionError as e:
    print("API failed:", e)     # retried 2x already
except AIBridgeError as e:
    print("Bridge error:", e)   # catch-all
